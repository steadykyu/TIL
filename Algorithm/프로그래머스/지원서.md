"간단하더라도 어떻게든 Web 서비스를 만들어보자. 그래야 여러 기술들의 존재 이유를 알고 더 질 좋은 지식을 얻을 수 있다."

5. (300자 내외, 최대 500자) 왜 개발자가 되고 싶으신가요?
```
내부적 이유로는 제가 "어떤 것의 원리나 문제를 이해하고 설명하거나 해결하는 것을 좋아하는 성격" 때문입니다. 이전의 회사에서 어떻게 데이터를 수집하고 가공했는지 자료 준비를 하고 발표하며 뿌듯함을 느끼게 해주는데 큰 도움을 주었습니다. 이런 성격은 서로가 이해한 내용을 설명하고 공유하며, 협업을 중요시하는 개발자의 모습에 어울리는 성격이라고 생각했습니다.

외부적 이유로는 여러 사람들에게 좋은 영향을 끼치는 일들을 비교하였을 때, 소프트웨어가 주는 영향력이 크다고 생각했기 때문입니다. 소프트웨어는 현재 사람들의 시간을 아껴주거나, 여러 분야에 편리함을 가져다주는 등 세상에 수많은 영향을 끼치고 있습니다. 저도 이 영향의 일부분이 되어 좋은 영향을 끼치며 경제 생활을 하는 한 명의 개발자가 되고 싶습니다. 
```

6. (300자 내외, 최대 500자) 웹 백엔드 분야로 진출하고자 결심한 이유는 무엇인가요?
```
이전까지 SQL을 사용하여 데이터를 가공하는 업무를 작업했습니다. 어떻게 하면 DB데이터를 빨리 처리하여 보내는지 팀원들과 회의하며 가공하는 작업이 적성에도 꽤 맞다는 생각이 들었습니다.  이때 서버 쪽 사람들의 일을 도와드리면서 서버에 데이터가 어떻게 수집되고 활용되는지 처음으로 관심을 가지게 되었습니다. 이러한 관심은 전반적인 개발 프로세스에 궁금증을 불러 일으켰고, 평소에 엑셀이나 DB 데이터를 자주 접하다 보니 디자인이나 웹을 역동적으로 만들어주는 CSS나 javascript가 저에겐 더 생소하게 느껴졌고 결과적으로 조금 더 익숙한 서버 사이드 개발에 진출해보고 싶다는 마음이 생겼습니다.  

또한 언젠가 실력을 좀 키우게 된다면, 대용량 데이터를 관리해보고 싶은 욕구가 있었습니다. 이는 백엔드 분야로 진출하려는 마음을 더 강하게 만들어 주었습니다.

```

자바의 정석 책을 읽으며 개념을 공부하고 최대한 도움 없이 문제를 풀려고 노력했습니다.

"알고리즘"
